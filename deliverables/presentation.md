# 📽️ Slide Deck: ShowTime Presentation

## **Slide 1: Title & Group Introduction**
- **Headline:** ShowTime: Premium Event & Ticketing System.
- **Sub-headline:** Enhancing the digital entertainment booking experience.
- **Presenters:** [Group Members]
- **Speaking Point:** Start by introducing the project name "ShowTime" and the members. Explain that this is a full-stack ticketing platform for movies, concerts, and more.

---

## **Slide 2: Identifying the Need**
- **Bullet Points:**
  - Need for intuitive seat selection.
  - Slow loading times on existing platforms.
  - Lack of a premium, immersive user interface.
- **Speaking Point:** Discuss why we built this. Highlight the gap in existing platforms (clunky UX/UI) and how ShowTime addresses it with a "Glassmorphism" design.

---

## **Slide 3: Key Features (The Wow Factor)**
- **Bullet Points:**
  - **Dynamic Home Dashboard:** Movies, Streams, Events.
  - **Visual Seat Selection:** Interactive theater grid.
  - **Multi-platform Auth:** Google Sign-In & JWT.
  - **Seamless Payments:** Integrated Razorpay checkout.
- **Speaking Point:** Briefly mention the core features that differentiate the project. Focus on the "Live Seat Mapping" as a technical highlight.

---

## **Slide 4: System Architecture (The Tech Stack)**
- **Diagram (to be drawn):** Controller -> Model -> View.
- **Bullet Points:**
  - **Frontend:** HTML5, CSS (Glassmorphism), Vanilla JS.
  - **Backend:** Node.js, Express.js.
  - **Database:** MongoDB Atlas (NoSQL).
  - **Deployment:** Vercel (Client), Render (API).
- **Speaking Point:** Explain why we chose Vanilla JS for performance and why MongoDB Atlas is perfect for scaling event data.

---

## **Slide 5: User & Admin Workflows**
- **Flow A:** User Discovery -> Selection -> Seating -> Payment.
- **Flow B:** Admin Dashboard -> Add Movie -> View Analytics.
- **Speaking Point:** Describe the dual-ended nature of the platform. Show how admins have full control over the inventory while users get a streamlined booking experience.

---
